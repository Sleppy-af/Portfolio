Why open-source software actually help us improve the world

12/04/2026 8 min read

We like to think of open-source softwares as a developer thing

* a github repo
* a side project
* a cool tools with 300 stars and readme that nobody cares about

But seeing it that way misses the point entirely . Open source is not just a development model, it's an infrastructure model for society. And when it works, it doesn't just improve software, it improves how the world functions.

Public money, public code

Let's start with something that should be obvious, yet somehow still it isn't. If taxpayers fund software, taxpayers should have access to it. Not as a bonus, not as a "nice to have". As a default. When governments build closed systems, they're effectively saying:

"Trust us. You don't need to worry, we have it"

But trust without transparency is fragile. Open-source software flips that model. It allows citizens, researchers and independent experts to verify, not just believe. That difference mattwrs more than we often admit. It's tje difference between: "We think it's secure" and "Anyone can audit this system and prove whether it is or not". That's not ideology. That's accountability

The compounding effect of shared effort

One of the most underated aspects of open source is how it compounds over time. A single project can start small, trivial and still become foundational. Take something as simple as curl. One tool with one original author. It's everywhere now: servers, phones, embedded systems. Not because corpos decided to push it globaly but because it was available, adaptable and reusable. This is how open source scales. One person builds something useful, others improve it, someone adapts it into a larger system and then another team integrates it into a larger system. And suddenly, somethung that started as a random 50 lines script becomes a monumental part of a global infrastructure. That's not an accident. That's a model working as intended

Open source as a force multiplier

In closed environments, progress is linear. In open ecosystems, it's random and that's exactly a good part. When code is available bugs can be found by anyone, features can be added by anyone, integrations can be built without permissions. This turn everyone into contributors, everyone can add something. And that changes everything. Because the biggest bottleneck in software isn't ideas, it's their execution. Open source removes that bottleneck by allowing everyone to repair the problems with all they know

Transparency is not the same as security

Let's be honest for a quick moment. Open source is often sold with a simple argument.

"More eyes = more secure software"

It sounds great. It's also incomplete. There are real-world examples where this didn't hold. Idk if you heard about xz-utils. For ppl that don't know, the xz project builded trust across developers for years with their fully open source code and then made a backdor which nobody would find out if not some random accident. Even tho the code was public, backdor still somehow made it in. It all proves one specific point. Openness allows inspection but it does not guarantee it happens. So no, open source is not magically secure. But the important part is, closed source doesn't solve this problem either, it only hides it. With proprltary systems, vulnerabilities can exist for years without anyone outside the vendor knowing. With open source, flaws are at least discoverable, still it doesn't elininate all the risks but it funtamentally changes the power which can detect and fix it.

The right to adapt

One of the most powerful and often overlooked feautures of open source is the right to fork. If a project stops evolving, you can continue it. If a company changes direction, you can easily preserve what mattered. If a tool no longer fits what you expect from it, you can change it all you wish. This creates resilience. We've seem ot repeatedly, projects forked after license changes, communities continuing abandomed tools, entire ecosystems rebuilt around shared foundations. THis is not fragmentation, this it evolution under user control. And when applied to public infrasturcture, it becomes critical, because governments, vendors and priorities change. Code should not disappear when they do.

Oepn source hangs beyond developers

Open source inpact goes far and beyond engineering teams and single programers. It affects education, security researches, journalism and publicy trust as much as devs. Students learns from real, open source system cuz they're easy to find. Systems can be investigated by journalists as much as they wish and transparency replaces the blind trust many companies build. It turns software from a black box into a shared artifact, something that can be studied, improved and questioned if needed

The bigger picture

Open source is not only about the code, it's about who has acces, control, and who gets to participate. It's about shifting from centralized ownership to shared responsibilities. We live in a world that is heavily dependent on software, and that shift matters... a lot.

Summary

Open source don't quarantee a better world, but it creates conditions where evolving is easier and the vision of better world is possible. More collaboration, more resilience and most importantly more transparency.
